# FX Trade Journal — 使い方ガイド

`docs/FX_Trade_Journal.xlsx` を Google Sheets で開いて、毎日のトレードを記録するためのテンプレート。

## 📋 構成

| シート | 役割 |
|---|---|
| **ダッシュボード** | 自動集計の KPI / 手法別・通貨ペア別パフォーマンス |
| **取引履歴** | 1 トレード = 1 行で記入。集計は自動 |
| **設定** | 開始残高・リスク%・pip 価値テーブル |

---

## 🚀 Google Sheets で開く手順 (3 分)

### Step 1: ファイルをダウンロード

GitHub から: <https://github.com/Apollo000007/fx-signal-monitor/raw/main/docs/FX_Trade_Journal.xlsx>

または、ローカルの `~/dev/fx-signal-monitor/docs/FX_Trade_Journal.xlsx` を使用。

### Step 2: Google Drive にアップロード

1. <https://drive.google.com/> を開く
2. **「+ 新規」 → 「ファイルのアップロード」**
3. `FX_Trade_Journal.xlsx` を選択

### Step 3: Google Sheets で開く

1. Drive 内で `FX_Trade_Journal.xlsx` を **右クリック**
2. **「アプリで開く → Google スプレッドシート」**

→ 初回オープン時に formula が自動再計算されます。

### Step 4: Google Sheets 形式に変換 (推奨)

xlsx のままだと一部機能が制限されるので、Google Sheets ネイティブ形式に変換:

1. 開いたスプレッドシートで **ファイル → Google Sheets として保存**
2. または **ファイル → コピーを作成 → 形式: Google Sheets**

これで `.xlsx` 接尾辞が外れ、完全な Google Sheets ファイルになります。

## ✏️ 毎日の記入手順

### 1. トレードしたら「取引履歴」シートを開く

### 2. 新しい行に以下を入力 (青色のセルだけ)

| 列 | 項目 | 入力例 | 必須? |
|---|---|---|---|
| B | **日付** | `2026/5/15` | ✅ |
| C | **通貨ペア** | ドロップダウンから選択 (USD/JPY 等) | ✅ |
| D | **手法** | TRIPLE / ORZ / PDHL / BOTH / CLAUDE | ✅ |
| E | **方向** | Long / Short | ✅ |
| F | スコア | 85 (signals.json の score) | 任意 |
| G | Entry 時刻 | `10:30` | 任意 |
| H | **Entry 価格** | `157.20` | ✅ |
| I | **SL 価格** | `156.80` | ✅ |
| J | **TP 価格** | `158.40` | ✅ |
| K | **ロット数** | `0.10` | ✅ |
| L | **決済価格** | `158.40` | (決済後) |
| M | **決済理由** | TP / SL / 手動 / 強制 / 建値撤退 | (決済後) |
| T | メモ | "東京時間、3R 到達" | 任意 |

### 3. 自動計算される列 (触らない)

| 列 | 項目 | 計算式 |
|---|---|---|
| A | 番号 | 自動連番 |
| N | SL pips | \|Entry − SL\| / pip サイズ |
| O | 計画 RR | \|TP − Entry\| / \|Entry − SL\| |
| P | 損益 pips | 決済方向に応じた損益 |
| Q | R-multiple | 損益 pips / SL pips |
| R | 累計 R | 全トレードの累積 R |
| S | 損益 ¥ | 損益 pips × ロット × pip 価値 |

→ 数字を入れた瞬間にすべて再計算されます。

### リスク管理セクション

- 最大連勝 / 最大連敗
- 最大ドローダウン
- 勝ち平均 R / 負け平均 R
- 現在の残高

| 手法 | 件数 | 勝率 | EV (R) |
|---|---|---|---|
| TRIPLE | 3 | 100% | +2.50 |
| ORZ | 5 | 40% | -0.30 |
| PDHL | 4 | 50% | +0.10 |
| BOTH | 0 | — | — |
| CLAUDE | 0 | — | — |

### 通貨ペア別パフォーマンス (15 ペア)

累計 R が **赤 → 白 → 緑** で色分け表示 (ヒートマップ)

## ⚙️ 設定を変更する

「設定」シートの **黄色いセル** だけ編集してください:

| セル | 項目 | 推奨値 |
|---|---|---|
| C3 | 開始残高 | 1,000,000 |
| C4 | 1 トレードリスク% | 0.5 |
| C5 | USD/JPY 換算レート | 月次で更新 (例: 155) |

`C5` (USD/JPY レート) は非 JPY ペアの円換算に使用するので、月初に最新値に更新するとより正確になります。

pip テーブル (B8:D23) は変更不要です。

## 🎨 視覚的機能

- 🟢 **勝ちトレード**: R-multiple 列が緑、決済理由 "TP" が緑背景
- 🔴 **負けトレード**: R-multiple 列が赤、決済理由 "SL" が赤背景
- 📊 **ヒートマップ**: 通貨ペア別累計 R が色グラデーション
- 🔄 **シマシマ行**: 偶数行が淡い紫で区別しやすく
- 🔒 **ペイン固定**: ヘッダーと最初の 2 列 (#、日付) はスクロールしても見える

## 💡 記入の Tips

### Tip 1: 一括コピペで楽できる

MT5 の取引履歴を CSV エクスポートしてコピペすれば、複数トレードを一気に記入できます。

### Tip 2: スコアは signals.json から

トレード時に Telegram に来たアラートメッセージのスコア (例: `スコア: 85/100`) をそのまま F 列に。

### Tip 3: 決済前は L, M 列を空欄に

ポジション保有中はトレードログを「途中」状態にしておき、決済したら L (決済価格) と M (決済理由) を入力。自動的に P (損益 pips) と Q (R-multiple) が計算されます。

### Tip 4: メモは具体的に

「東京時間、3R 到達」「NY 引け間際でスプレッド広がってた」など、後で振り返れる情報を残す。これが戦略改善の原資になります。

### Tip 5: 週末に振り返り

毎週日曜にダッシュボードを見て、以下を確認:
- どの手法が勝っているか
- どのペアが勝っているか
- 最大ドローダウンは許容範囲か

## 🔄 サンプルデータの削除

xlsx には 3 件のサンプルが入っています (行 2-4)。

**実トレードを始めたら、サンプル 3 行を選択して削除** してください:
1. 行 2-4 を選択
2. 右クリック → 「行を削除」または「内容を削除」

集計式は自動的に空白セルを無視するので、削除しても問題ありません。

## 🛠 トラブルシューティング

### Q. 損益が計算されない (列 P / Q が空のまま)
A. **C 列 (通貨ペア) と E 列 (方向) と H/I 列 (Entry/SL) と L 列 (決済価格) すべて入力されているか** 確認してください。1 つでも欠けると計算されません。

### Q. 円換算がおかしい
A. 「設定」シート C5 の USD/JPY レートを最新値に更新してください。

### Q. ドロップダウンが出ない
A. Google Sheets では xlsx の検証ルールが部分的に動かない場合あり。「ファイル → Google Sheets として保存」で変換すれば確実に動きます。

### Q. ダッシュボードの数字が更新されない
A. **ファイル → 最新の情報に更新** または ⌘+R でブラウザを再読込してください。

### Q. もっと多くの行が必要 (500 件超え)
A. 取引履歴シートは 500 行で固定です。さらに必要な場合は最後の行で「形式 + 数式」をコピーして拡張してください。または、月別に新しいシートを作って分割管理することを推奨。

## 📈 推奨ワークフロー

```
朝 (取引前):  ダッシュボードで現在残高・累計 R を確認
              許容ドローダウンを意識
              ↓
日中:        トレードしたら「取引履歴」に即記入
              (Entry 直後と決済後の 2 回)
              ↓
夜:          ダッシュボードで当日の R を確認
              メモ欄に学びを書く
              ↓
週末:        手法別・ペア別の傾向を分析
              負け続けている手法は一旦停止検討
```
