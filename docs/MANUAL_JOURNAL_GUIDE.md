# FX Manual Trade Journal — 手動トレード戦績スプレッドシート

`docs/FX_Manual_Trade_Journal.xlsx` — **手動トレード専用**、5 手法を個別に追跡して比較するためのジャーナル。

## 🎯 これを使う理由

| | 自動 EA (FXSignalEA) | 手動トレード (このシート) |
|---|---|---|
| 戦績記録 | EA ログから自動 | **手で書く必要あり** ← これ |
| 計画 vs 実 | 同一 | **乖離が起きる**、それを記録 |
| トレード評価 | なし | **S/A/B/C/D で自己評価** |
| 学びノート | なし | **手法別に蓄積** |

→ 「自動売買は EA が淡々と。**手動で挑戦したい時はこのシートで分析**」という棲み分け。

## ✏️ 各手法シートの記入方法

### 上部の自動ダッシュボード (行 1-15)

最初の 15 行は **触らない領域** です。トレード履歴を書き込むと自動更新されます:

```
TRIPLE  · 手動トレード履歴
3 手法すべてが合意したときのみ発火。Backtest で唯一の +EV...
─────────────────────────────────────────────────────────────────
📊 集計サマリ            🏅 トレード評価分布      📝 TRIPLE で学んだこと
件数         5            S ランク  2  40%        (ここに手法別の学び・観察を書く)
勝率        80%           A ランク  2  40%        ・東京時間の発火は勝率高い
平均RR      2.50          B ランク  1  20%        ・スコア 85+ で取ると安定
PF          3.20          C ランク  0   0%        ・SL は signal の値より浅めが良い
期待値    +1.250R         D ランク  0   0%
累計R     +6.25R
累計¥    +¥62,500
```

### トレード履歴の入力 (行 18 以降)

行 17 がヘッダー、行 18 から実際の取引データを記入します。

#### 入力する列 (青背景)

| 列 | 項目 | 例 | 説明 |
|---|---|---|---|
| B | **日付** | `2026/5/15` | エントリー日 |
| C | **通貨ペア** | USD/JPY (dropdown) | |
| D | **方向** | Long / Short (dropdown) | |
| E | スコア | 85 | アラート時の signal score |
| F | **計画 Entry** | 157.20 | シグナルの推奨価格 |
| G | **実 Entry** | 157.25 | 実際の約定価格 |
| H | **計画 SL** | 156.80 | シグナルの推奨 SL |
| I | **実 SL** | 156.80 | 実際に設定した SL |
| J | **計画 TP** | 158.40 | シグナルの推奨 TP |
| K | 実 TP | 158.40 | 実際に設定した TP |
| L | **ロット** | 0.10 | |
| M | **決済価格** | 158.40 | (決済時に記入) |
| N | **決済理由** | TP / SL / 手動 / 強制 / 建値撤退 | (dropdown) |
| U | **トレード評価** | S/A/B/C/D (dropdown) | 後述 |
| V | **学び / 反省** | "東京時間は窓開け含み" | 自由テキスト |

#### 自動計算される列 (触らない)

| 列 | 項目 | 計算式 |
|---|---|---|
| A | 番号 | 自動連番 |
| O | 実 SL pips | \|実Entry − 実SL\| / pip サイズ |
| P | 計画 RR | \|計画TP − 計画Entry\| / \|計画Entry − 計画SL\| |
| Q | 損益 pips | 決済方向に応じた損益 |
| R | R-multiple | 損益 pips / 実SL pips |
| S | 累計 R | 累積 |
| T | 損益 ¥ | 損益 pips × ロット × pip価値 |

## 🏅 トレード評価ランクの基準

`U 列` に **S/A/B/C/D** を入れて、トレードの "質" を自己評価します。
**勝ち負けと関係なく、規律と実行力で評価** するのがポイント:

| ランク | 意味 | 例 |
|---|---|---|
| **S** | 完璧 | 計画通りエントリー → 計画通り決済 → 利益確保 |
| **A** | 良い | 多少のズレはあったが、最終的に利益 |
| **B** | 普通 | 微益・微損で終了。学びは少ない |
| **C** | 悪い | 損失だが、なぜ負けたか理由が分かる |
| **D** | 失敗 | 規律違反 (ロット過大 / SL ずらし / 早すぎる利確 / FOMO エントリー) |

> **D ランクのトレードが連続したら、休んでメンタル整理する** のがプロの鉄則です。

## 📊 ダッシュボードの読み方

「📊ダッシュボード」シートで 5 手法を横並び比較できます:

```
指標            TRIPLE   ORZ    PDHL   BOTH   CLAUDE
件数            5        12     8      3      5
勝率           80.0%    58.3%  50.0%  33.3%  60.0%
平均RR (計画)  2.50     1.80   1.85   1.50   2.20
PF             3.20     0.85   0.95   0.40   1.50
期待値 (R)    +1.250   -0.150 +0.025 -0.400 +0.500   ← 緑=+EV, 赤=-EV
累計 R        +6.25    -1.80  +0.20  -1.20  +2.50    ← ヒートマップ
累計損益 ¥   +¥62,500 -¥4,500 +¥800 -¥6,000 +¥12,500
最大 DD (R)   0.80     1.50   2.20   3.50   1.00
勝ち平均 R    +2.10    +1.40  +1.80  +1.20  +2.30
負け平均 R    -1.05    -1.20  -1.05  -1.50  -1.10
評価 S 件数   2        1      0      0      1
評価 D 件数   0        2      1      2      0
```

**緑 = +EV (期待値プラス)、赤 = -EV (期待値マイナス)** で色分けされます。

下部に「全体サマリ」: 全手法合計の件数・累計R・累計損益・現在残高。

## 💡 推奨ワークフロー

### 朝 (取引前)

1. ダッシュボードを開く → 自分の現在残高と各手法の累積 R を確認
2. 「-EV の手法 (赤) は今日は取らない」と決める
3. Telegram 通知で来た TRIPLE シグナルだけ集中

### トレード中

1. シグナルが来たら、該当手法のシート (例: TRIPLE) を開く
2. 計画 Entry/SL/TP を **行 18 以降に書く**
3. 実 Entry が違うなら G 列に実際の値を入力
4. ロット、計画 RR が自動計算されるので **「計画 RR が 2 未満なら見送り」** など事前判断

### 決済後

1. **M 列 (決済価格) と N 列 (決済理由) を入力**
2. R-multiple と損益¥が自動計算
3. **U 列でトレード評価 (S/A/B/C/D)** を入れる
4. **V 列に学び/反省** を必ず書く (1 行でも OK)

### 週末

1. ダッシュボードで 5 手法の累計 R を比較
2. **-EV の手法は来週は休止** を検討
3. 各手法シート上部の「学びメモ」を読み返して傾向を整理

## 🔥 プロの使いこなし術

### Tip 1: 計画 vs 実 の乖離を見る

D ランクのトレードを見直すと、しばしば **「計画 Entry と実 Entry がズレている」** ことが分かります:
- 「Entry を待ちきれずに早めに入った」
- 「FOMO で計画と逆方向に入った」

このパターンが多い手法は、**自分のメンタル特性と合っていない** 可能性大。

### Tip 2: S ランクが多い手法 = 自分の強い手法

S ランクのトレードが多い手法は、**自分のスキルと相性が良い** 手法です。
そこに集中投資 (ロット増) するのが理にかなっています。

### Tip 3: 同じ pair × method を観察

例えば「TRIPLE × USD/JPY」だけで 3 連勝したら、その組み合わせが **黄金パターン** かもしれません。
学びメモに「USDJPY TRIPLE は東京時間に強い」など蓄積していくと、将来のフィルタリング材料に。

### Tip 4: 月末に "棚卸し"

毎月末に:
- 累計 R 最大の手法 → 来月はメイン
- 累計 R 最小の手法 → 来月は休止 or リスク 1/2
- D ランクの内訳分析 → 規律違反のパターン発見

## 🛠 トラブルシューティング

### Q. 損益が計算されない
A. 以下を確認:
- C 列 (通貨ペア) と D 列 (方向) と G 列 (実 Entry) と I 列 (実 SL) と M 列 (決済価格) **すべて入力済み** か
- 1 つでも空欄だと計算停止

### Q. 計画 RR が出ない
A. F (計画 Entry), H (計画 SL), J (計画 TP) すべて必要。1 つでも空だと計算不可。

### Q. ダッシュボードの「最大 DD」が変
A. 累計 R が一度もマイナスにならないと DD = 0。実トレード進めば自動的にちゃんと出ます。

### Q. シートを 1 つだけ削除したい
A. 使わない手法のシートは右クリック → 「シートを削除」で OK。
ただしダッシュボードが該当手法を参照しているので、削除する手法の列はダッシュボード上で `#REF!` になります (無視可)。
