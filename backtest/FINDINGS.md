## 結論 (TL;DR)

1. **`TRIPLE` (3 手法合意) のみが正期待値**。構造的 TP が最良 (PF 3.42)
2. **他 4 手法は TP を変えても救えない**。シグナル品質に課題
3. **TP を 3R 固定にすると ORZ/PDHL/CLAUDE の PF は改善 (0.58→0.82 等) するが、まだ <1**
4. **本番運用は TRIPLE 一本** + 元の構造的 TP がベスト

## 観察

### TRIPLE がトップの理由

- 3 手法 (ORZ+PDHL+Claude) **すべてが同方向で alert を出した時のみ** 発火
- 合意条件が非常に厳しいので **サンプル数は少ない (5 件) が質が高い**
- 構造的 TP が偶然 2-3R 近辺になっていて、勝率 60% × RR 2.6R で PF 3.42

### ORZ の高勝率なのに負ける理由

- 勝率 59% は方向判定は正しい
- でも構造的 TP (近接レジサポ) が SL より近い場合が多い → avg_win < avg_loss
- TP を 3R に伸ばすと届かなくなり、勝率が 26% に低下
- 25.9% × 3R = ブレークイーブン手前で、PF 0.82 にとどまる

### 救えない理由

- ORZ/PDHL/CLAUDE は **「方向は当てるが、利益が小さい」** タイプ
- 短いリテスト距離で利確 → 統計的に小利確 + 大損切のパターン
- これを改善するには **シグナル品質を上げる** か **エントリー精度を上げる** しかない

## 結論を踏まえた運用判断

### 短期 (今すぐ)

✅ **MT5 EA は `UseTriple=true` のみで運用** (既にそう設定済み)
✅ Paper Trade は全 5 手法走らせて Live データを蓄積
✅ アラート通知も TRIPLE のみに絞っても良い (Telegram のノイズ削減)

### 中期 (1-2 週間後)

- Paper Trade で 1 週間分蓄積したら、Live と Backtest を比較
- TRIPLE が Live でも +EV なら少額 Live トレード開始 (EA + Demo → Live)
- 他手法のデータも見て、Backtest と Live で差があるか確認

### 長期 (戦略改善)

- ALERT_THRESHOLD = 75 を 80, 85 に上げて精度向上を狙う
- スコア配点を再設計 (例: ★15M トリガー +20 → +30)
- 個別ペアで強弱を分析 (ヒートマップで明示済み)
- TRIPLE のサンプル数が少なすぎる → 4 手法以上の合意モードを追加検討
